사전 세팅 (상품 세팅 페이지 구성)

1. 어떤 상품을 작업할지 선택하고 해당 상품에대해 클로드 api를 이용해 카페 글 본문과 댓글 작성시 필요한 내 상품에 대한 상세 정보와 댓글에 들어갈 내 상품 링크를 저장해두고 관리할 신규 페이지(상품 세팅)가 있어야합니다. 
먼저 해당 페이지에서는 상품관리 페이지 https://scorp274.com/products 에 있는 상품 데이터를 가져와서 작업할 상품을 추가하고싶다. 썸네일과 상품명만 상품 관리 페이지에 있는 순서대로 가져오면 됩니다. 마치 마케팅 - 카페  - 상품 관리 탭에 있는 마케팅 상품 추가와 똑같은 기능과 ui를 만들어주시면 됩니다.
지금은 카페만 하지만 추후 블로그도 자동화 작업을 할 것 이기에 상품 세팅 페이지에서 체크박스로 블록그에 작업할지 카페에 할지 아니면 둘다할지 체크를 통해 해당 상품의 작업 권한을 부여하고싶다
그리고 제일 중요한 해당 상품에 대한 상세 정보와 디테일한 장단점 특징, 우리 제품의 cv(핵심주장)등 클로드 api 프롬프트에 우리 제품을 더 잘 알아서 현실적으로 쓸 수 있도록 해야하고 댓들에 작성될 우리 제품의 마케팅 링크까지 저장되어야합니다. 우리 제품의 상세정보 입력란은 1, 우리제품명 2, 우리 제품의 핵심 3, 우리 제품의 서브 핵심 4. 우리 제품 사이즈& 무게 5. 타사 제품과 차별점 6. 해당 제품의 유명 브랜드들 7. 기존 시장의 문제점 8. 우리 제품의 가격  9. 기존 시장의 평균 가격 10. 고객의 예상 연령대 11. 고객의 예상 성별 12. 기타 추가 특이사항(선택) 이렇게 입력 필드 항목을 만들어줘 마지막 2,3,5,7,12 번 항복의 답변 입력 칸은 좀 내용이 많은 수 있으니 넓게 만들어줘 12번 항목 제외하고는 모두 필수야 그리고 마케팅 링크는 해당 상품의 마케팅 링크를 넣을 수 있는 필드를 만들어줘 이것도 필수야 그리고 이걸 모두 모달로 만들어서 수정 버튼을 눌러서 모달에서 작업할 수 있도록 해야해.

2. 상품별로 키워드 관리 버튼을 만들고 키워드 별로 대안성, 정보성 키워드 별로 분류를 해둔다. (기존 마케팅 - 카페에 제품별 타켓키워드를 동기화해 온다 그리고 맨처음 동기화 한 후 모두 미분류로 바꾸고 하나씩 분류를 작업자가 직접 분류하고 싶다. 필요하면 신규 키워드 추가와 삭제도 가능하게 하고싶고 해당 데이터는 마케팅 - 카페 부분과 동기화 되지 않아야합니다.)

3. 상품별로 레퍼런스 관리 버튼을 만들고 대안성, 정보성 각각 분류에 맞는 본문+댓글 레퍼런스를 분류해둔다. (기존 마케팅 - 카페에 레퍼렌스 관리에 있는 레퍼런스들과 동기화해 온다. 그리고 이때 마케팅 - 카페에 있던 레퍼런스 상세 정보 https://scorp274.com/marketing/reference/ 이 링크의 구성을 그대로 가져와서 보여줘야한다. 단 여기도 마찬가지로 신규 레퍼런스를 추가할 수 있고 기존 레퍼런스를 수정 작제할 수 있지만 자동화 쪽에서 작업한 내역은 마케팅 - 카페의 레퍼런스에 동기화 되지 않는다, 맨처음 동기화 한 후 모두 미분류로 바꾸고 하나씩 분류를 작업자가 직접 분류하고 싶다.)


프롬프트 관리

0. 프롬프트의 템플릿을 대안성과 정보성으로 미리 만들어두고싶다.
프롬프트 템필릿은 프롬프트 관리내에 프롬프트추가 버튼 왼쪽으로 템플릿 추가 버튼을 누르고 사용자 프롬프트 탬플릿을 추가할 수 있는 모달을 열고 거기서 어떤 분류인지 선택하고 프롬프트 이름을 설정하고 사용자 프롬프트 템플릿을 입력하고싶다. 이때 중요한건 상품 세팅 페이지에서 추가한 상품 상세 정보부터 마케팅 링크 까지의 값을 머리말로 변수화 해서 버튼을 입력 필드 위에 변추 추가 버튼과 드롭박스로 추가할 변수명을 보여주고 사용자 프롬프트 템플릿에 아주 쉽고 빠르게 추가할 수 있도록 하고싶다. 이때 중요한건 템플릿 본문에서 벼수 부분은 버튼처럼 표시하고싶다.
그리고 저장하고 나서 프롬프트 목록에서 타입을 템플릿과 실제 상품이라는 타입 필터를 만들어서 템플릿 부분을 클릭했을때만 나오도록 하고싶다. 템플릿 목록에서 보여야 할 정보는 템플릿 분류(대안성 정보성), 템플릿 명, 수정 버튼 삭제 버튼, 복제 버튼이 필요하다

1. 프롬프트는 상품 별로 관리하고싶다.
2. 프롬프트 추가 버튼을 누르면 제일먼저 어떤 상품에 대한 프롬프트인지 자동화에 추가된 상품목록을 드롭박스로 선택하고싶다.
3. 프롬프트 타입이라는 선택명을 키워드 분류로 바꾸고 해당 드롭박스의 내용을 대안성, 정보성 이렇게 두개로 하고싶다.
4. 시스템 프롬프트는 기존과 동일하게 하고싶다
5. 사용자 프롬프트 템플릿 부분에서는 먼저 기존 저장된 템플릿 중 어떤 분류의 템플릿을 사용할지 분류를 드롭박스에서 먼저 선택하고 그러면 두번째 드롭박스가 나타나면서 해당 분류도 만들어진 탬플릿 중에 실제로 어떤 템플릿을 사용할지 선택한고 해당 선택한게 입력 필드에 표시되도록 한다 (함수 버튼 뱃지도 똑같이 표현)
6. 템플릿 생성때와 마찬가지로 입력 필드위에 드롭박스로 상품 상세 정보와 마케팅링크에 대한 변수를 선택하고 추가할 수 있도록 해야한다. 
7. 변수 추가 버튼 옆에 이미지 생성 버튼을 추가해서 템플릿 제일 하단에 "위 내용을 정확하게 참조해서 1, 제품 파손, 불량 등 부정적인 실제 사진같은 이미지 2. 실제 한국사람이 고통스러워 하고있는 실제 사진같은 이미지 3. 해당 제품의 실제 사용하는 것 같은 이미지 이렇게 총 3장을 만들어줘" 라고 넣어줘
8. 추후 생성된 해당 이미지를 본문 글 별로 저장해서 카페 본문에 올려야한다.
9. Temperature와 맥스 토큰은 그대로 유지한다.
